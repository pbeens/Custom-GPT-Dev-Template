# Prompt Engineering Checklist

Use this checklist when reviewing or creating new prompt versions.

1.  **Persona/Role**: Does the prompt clearly state who the AI is? (e.g., "You are an expert functionality tester...")
2.  **Context**: Does it provide necessary background info?
3.  **Task**: Is the primary task clearly defined?
4.  **Constraints**: What should the AI *not* do? (e.g., "Do not use emojis", "Keep answers under 50 words").
5.  **Output Format**: Is the desired output format specified? (e.g., JSON, list, markdown table).
6.  **Examples (Few-Shot)**: Are there examples of User Input -> Correct Assistant Output? This is the most powerful way to improve performance.
7.  **Tone**: Is the tone specified? (Professional, casual, witty, Socratic).
8.  **Security**: [MANDATORY] Is the Security Policy included at the end?
